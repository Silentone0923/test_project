心得與反思：打破框架的旅程

從「社會組」到「開發者」

一直以來，「文組不懂電腦」的標籤似乎都貼在我們身上。但透過這次 Python 的學習之旅，我發現程式語言其實也是一種「語言」。就像我們學習諮商技巧是為了理解人的內心運作，學習程式則是為了理解電腦的思考邏輯。這段跨領域的學習不僅僅是為了彌補高中的遺憾，更是讓我學會用更具邏輯性的視角來分析問題。

與 Bug 共處的修煉

在製作「終極密碼」和「TRPG」的過程中，最常面對的不是靈感枯竭，而是紅通通的錯誤訊息 (Error Message)。起初會感到挫折，但逐漸地，我將 Debug 的過程視為一種「諮商」。我需要耐心地傾聽（看Log）、找出核心問題（定位錯誤）、並嘗試不同的介入方式（修改程式碼）。每一次成功運行的瞬間，帶來的成就感是無可比擬的。

未來展望

雖然目前的作品還很稚嫩，且許多部分仰賴 AI 的協助，但這些都是成長的養分。未來我希望能將心理學知識與程式結合，或許是開發一個簡單的心理測驗 App，或是利用數據分析來探討心理議題。這條 Python 之路，才正要開始。
